## 1. Multi‑user & accounts (new)
- **Login / register / logout**, hand‑rolled on Flask sessions + werkzeug password hashing —
  no heavyweight dependencies. First account becomes **admin** and inherits the existing
  single‑user collection/watchlist automatically.
- **Per‑user isolation:** every collection and watchlist row is scoped to its owner. Verified
  by test that two users can't see each other's data.
- **CSRF protection** on every state‑changing request, **rate limiting** on login/register
  (brute‑force guard), secure cookie flags, and a 5 MB upload cap.
- **Leaderboard + public collection profiles** (opt‑in): share `/u/yourhandle`. Purchase
  prices stay private — only market value shows publicly.

## 2. Deploy readiness (Render)
- **Postgres backend option** alongside SQLite — flip on with a `DATABASE_URL` env var. A
  translation adapter rewrites the SQL at runtime so the app code stays unchanged. SQLite is
  still the local default.
- `wsgi.py` (gunicorn/waitress entry), `render.yaml` blueprint (web + Postgres + nightly cron
  worker), `DEPLOY.md`, `.env.example`, `.gitignore`.
- Migration script to copy the local SQLite data into Postgres when you're ready.

## 3. Ops hardening
- **Automatic pre‑crawl DB backups** (rolling, keeps last 14) so a bad crawl can't cost you
  data.
- **Cross‑process crawl lock:** the "Run Crawl" button greys out whenever *any* crawl is
  running — app, scheduler, or manual — so you can't accidentally double‑crawl and corrupt a
  day's numbers (the old "truncation" mystery).
- **Rotating app logs** + optional error reporting (Sentry) that stays off unless configured.
- **Speed report after every crawl**, now with true parallel‑vs‑wall‑clock timing.

## 4. UI / UX you requested
- New‑user **feature tour** (8‑step walkthrough) on first login; re‑runnable from Settings.
- "Run Crawl" moved to the **Crawler** tab (renamed from History); speed‑report box beside it.
- **What‑is‑FangTrack** page: live vendor/listing counts, source‑confirmation guidance
  ("always confirm CB with the seller"), full rarity‑tier legend (Mythic/Legendary…). Removed
  all "start a crawl" instructions since crawls are on a timer for users.
- **Collection spreadsheet/CSV uploader**; imported your tarantula sheet and cleaned the notes.
- "Baseball card" renamed to **"spider card"** everywhere.
- **Leaderboard** moved left of "What is FangTrack?" in the header.
- **Sale badges** on deals (SALE / HOLIDAY / BOGO).
- Mobile responsiveness for nav and wide tables.

## 5. Data quality
- Listing **descriptions** now captured and cleaned during crawl (HTML stripped, capped).
- Better **species canonicalization**: more common‑name aliases, genus coverage, and a fix so
  localities like "sp. New Guinea" no longer collapse into a bare "sp."
- Fixed the **dead "biggest price drop" link** on the dashboard.

## 6. Weekend self‑review (independent pass)
Ran a separate review agent over everything above. It confirmed the important guarantees are
clean (no cross‑tenant data leak, no private‑price leak, CSRF fails safe). Fixed:
1. **[HIGH]** A Postgres‑only crash on the admin "set source policy" action — would have passed

2. Open‑redirect after login via a `/\` trick — closed.
3. Locality names collapsing during normalization — fixed (see §5).
4. Rate limiter could be bypassed with a spoofed forwarding header — now reads the real client
   IP that Render sets.
5. Force‑logout via a hostile image tag — logout is now a proper POST.
